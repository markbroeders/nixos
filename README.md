# nixos

Personal NixOS system configuration, flake-based, with home-manager wired in
as a NixOS module (one `nixos-rebuild` deploys both system and user config —
no separate `home-manager switch` step).

Currently defines a single machine, `wheeler` (host directory `hosts/asus`).

## Repo layout

- `flake.nix` — inputs (nixpkgs, home-manager, claude-code-nix) and the
  `nixosConfigurations.wheeler` definition.
- `hosts/asus/` — machine-specific config: `default.nix` (user account,
  imports) and `hardware-configuration.nix` (disks/filesystems, generated by
  the installer — don't hand-edit, don't copy to a different machine).
- `modules/` — system-wide config: packages, desktop, fonts, services.
- `home/` — user-scoped config via home-manager: shell, git, emacs,
  programming tools.
- `templates/` — flake templates for scaffolding new projects (see
  `templates/INSTRUCTIONS.md`).
- `MAINTENANCE.md` — day-to-day command cheat sheet (rebuilding, updating
  inputs, rolling back, garbage collection).

## Setting up on the machine this config targets (`wheeler`)

1. Clone the repo to `~/nixos`:
   ```bash
   git clone https://github.com/markbroeders/nixos.git ~/nixos
   cd ~/nixos
   ```
2. Make sure flakes are enabled (already the case if you're rebuilding from
   a checkout that has this set in `hosts/asus/default.nix` via
   `nix.settings.experimental-features`; on a fresh install you may need to
   enable flakes in `/etc/nixos/configuration.nix` once first, or pass
   `--extra-experimental-features 'nix-command flakes'`).
3. Apply the config:
   ```bash
   sudo nixos-rebuild switch --flake .#wheeler
   ```
4. Set a password for the `mark` user if this is a fresh install:
   ```bash
   passwd mark
   ```

See `MAINTENANCE.md` for the full set of day-to-day commands (building
without activating, boot-only activation, rollbacks, updating flake inputs,
garbage collection).

## Setting up on a *different* machine

This config is tied to one machine's hardware. To reuse it elsewhere:

1. Generate hardware config for the new machine:
   ```bash
   nixos-generate-config --show-hardware-config > hosts/<new-host>/hardware-configuration.nix
   ```
2. Copy `hosts/asus/default.nix` to `hosts/<new-host>/default.nix` and adjust
   as needed (username, `system.stateVersion` — see the warning comment in
   that file before changing it).
3. Add a `nixosConfigurations.<new-host>` entry in `flake.nix` pointing at
   `./hosts/<new-host>`.
4. `sudo nixos-rebuild switch --flake .#<new-host>`.

## Manual steps not covered by this repo

A couple of things are intentionally kept out of version control and need
to be set up by hand after cloning:

- **`gh` GitHub CLI auth** — run `gh auth login`, then `git push`/`git pull`
  over HTTPS will authenticate via the credential helper already configured
  in `home/programming.nix` (`gh auth git-credential`).
- **Emacs org-gcal secrets** — `~/.emacs.d/secrets.el` holds the Google
  OAuth client id/secret and calendar id mappings for `org-gcal`. It's not
  tracked in git (see `home/emacs.nix`); create it manually if you use the
  calendar sync feature (see the "Org Gcal Calendar" section of
  `home/config/emacs/emacs.org` for the variables it needs to set).
