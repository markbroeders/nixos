** Package management
This configuration uses Emacs' built-in package manager (=package.el=) together
with =use-package= (built into Emacs since version 29). We point =package.el= at
the GNU, NonGNU and MELPA archives, which between them provide essentially every
package referenced below as a /released/ version that is compatible with the
Emacs you are running.

Setting =use-package-always-ensure= to =t= means each =(use-package ...)= form
will install the package automatically if it is missing. Built-in packages (such
as =org=, =eldoc=, =flymake=, =which-key= and =modus-themes= on Emacs 30) are
marked with =:ensure nil= so they are configured but never downloaded.

#+begin_src emacs-lisp

  (require 'package)

  ;; Package archives. MELPA carries the bulk of community packages; the GNU and
  ;; NonGNU ELPA archives carry the rest (and are required for some dependencies).
  (setq package-archives
        '(("gnu"    . "https://elpa.gnu.org/packages/")
          ("nongnu" . "https://elpa.nongnu.org/nongnu/")
          ("melpa"  . "https://melpa.org/packages/")))

  ;; Initialise the package system.
  (package-initialize)

  ;; Refresh the archive contents the first time, so a fresh checkout can install
  ;; everything without you having to run `package-refresh-contents' by hand.
  (unless package-archive-contents
    (package-refresh-contents))

  ;; use-package is bundled with Emacs >= 29.
  (require 'use-package)

  ;; Install packages automatically unless a declaration says `:ensure nil'.
  (setq use-package-always-ensure t)

  ;; MELPA rebuilds its packages daily, so a cached `archive-contents' can point
  ;; at a tarball that has since been removed -- which shows up as a 404 ("Not
  ;; found") when installing. If an install fails, refresh the archives once and
  ;; retry, so a stale index heals itself instead of breaking startup.
  (defun mb/package-install-with-refresh (orig-fn pkg &rest args)
    "Around-advice for `package-install': refresh archives once and retry on error."
    (condition-case err
        (apply orig-fn pkg args)
      (error
       (message "Installing %s failed (%s); refreshing archives and retrying..."
                pkg (error-message-string err))
       (package-refresh-contents)
       (apply orig-fn pkg args))))
  (advice-add 'package-install :around #'mb/package-install-with-refresh)

#+end_src

** Performance Improvements
[[https://emacs-lsp.github.io/lsp-mode/page/performance/][source]]

The garbage-collection threshold is now managed in =early-init.el= (kept very
high during startup, restored to ~100mb afterwards), so we only need to bump the
amount of data Emacs reads from a subprocess here — the Emacs default of 4k is
far too small for language-server responses, which are often in the 800k–3M
range.

#+begin_src emacs-lisp

  ;; Increase the amount of data which Emacs reads from the process
  (setq read-process-output-max (* 1024 1024)) ;; 1mb

#+end_src

** Theme
Modus themes https://protesilaos.com/emacs/modus-themes#h:1af85373-7f81-4c35-af25-afcef490c111

The Modus themes ship with Emacs, but on Debian they are only placed on
=custom-theme-load-path= (so =load-theme= works) and *not* on =load-path= (so
=(require 'modus-themes)= fails). We therefore just load the theme directly,
rather than going through =use-package=, which would try to =require= the
feature and error out with "Cannot load modus-themes".

If you want the theme's customization API (=modus-themes-toggle=, the
=modus-themes-*= options, palette overrides, etc.), install the standalone
package instead — replace the =load-theme= call below with:
=(use-package modus-themes :ensure t :config (load-theme 'modus-vivendi t))=

* KEYBOARD
** Which Key
`which-key' is an Emacs package that displays available keybindings in a popup window whenever you partially type a key sequence. This is particularly useful for discovering commands and shortcuts, making it easier to learn Emacs and improve your workflow. It helps users remember key combinations and reduces the cognitive load of memorizing every command.

Since Emacs 30, =which-key= is part of Emacs itself, so we enable and configure
it with =:ensure nil=.

#+begin_src emacs-lisp

  (use-package which-key
    :ensure nil
    :init
      (which-key-mode 1)
    :diminish
    :config
    (setq which-key-side-window-location 'bottom
  	  which-key-sort-order #'which-key-key-order-alpha
  	  which-key-allow-imprecise-window-fit nil
  	  which-key-sort-uppercase-first nil
  	  which-key-add-column-padding 1
  	  which-key-max-display-columns nil
  	  which-key-min-display-lines 6
  	  which-key-side-window-slot -10
  	  which-key-side-window-max-height 0.25
  	  which-key-idle-delay 0.8
  	  which-key-max-description-length 25
  	  which-key-allow-imprecise-window-fit nil
  	  which-key-separator " → " ))

#+end_src

** Evil / VIM motions everywhere
Evil emulates the keybindings in Emacs and allows for extensive [[https://evil.readthedocs.io/en/latest/index.html][customization]]. Below is a pretty standard and simple configuration. Since =Evil= does not cover all the parts of Emacs, we use an extra package called [[https://github.com/emacs-evil/evil-collection][evil-collection]] which is a collection of additional keybindings for these parts that are not covered by Evil.

#+begin_src emacs-lisp

  (use-package evil
    :init
    (setq evil-want-integration t)
    (setq evil-want-keybinding nil)
    (setq evil-want-C-u-scroll t)
    (setq evil-want-C-i-jump nil)
    :config
    (evil-mode 1)
    (define-key evil-insert-state-map (kbd "C-g") 'evil-normal-state)
    (define-key evil-insert-state-map (kbd "C-h") 'evil-delete-backward-char-and-join)

    ;; Use visual line motions even outside of visual-line-mode buffers
    (evil-global-set-key 'motion "j" 'evil-next-visual-line)
    (evil-global-set-key 'motion "k" 'evil-previous-visual-line)
    ;; Same for up/down keys
    (define-key evil-normal-state-map (kbd "<down>") 'evil-next-visual-line)
    (define-key evil-normal-state-map (kbd "<up>") 'evil-previous-visual-line)

    (evil-set-initial-state 'messages-buffer-mode 'normal)
    (evil-set-initial-state 'dashboard-mode 'normal))

  (use-package evil-collection
    :after evil
    :init
    (setq evil-collection-company-use-tng nil)  ;; Is this a bug in evil-collection?
    :config
    (evil-collection-init))

  ;; Using RETURN to follow links in Org/Evil
  ;; Unmap keys in 'evil-maps if not done, (setq org-return-follows-link t) will not work
  (with-eval-after-load 'evil-maps
    (define-key evil-motion-state-map (kbd "SPC") nil)
    (define-key evil-motion-state-map (kbd "RET") nil)
    (define-key evil-motion-state-map (kbd "TAB") nil))
  ;; Setting RETURN key in org-mode to follow links
  (setq org-return-follows-link  t)
#+end_src

* COMPLETION FRAMEWORK

** Corfu
[[https://github.com/minad/corfu][Corfu]] is a lightweight Emacs package designed to enhance in-buffer completion. It provides a polished user interface for displaying completion candidates using child frames, and it integrates with Emacs's existing completion system. Corfu focuses on a clean, modern completion experience, aiming for performance and reliability while integrating with other completion-related packages like Cape. This part of my configuration was taken from [[https://github.com/garyo/emacs-config/blob/main/lisp/init-completion.el][this repo]].

The MELPA recipe for Corfu bundles its extensions (such as =corfu-history=), so
=corfu-history= below can be loaded with =:ensure nil=.

#+begin_src emacs-lisp

  ;;; In-buffer completion (using complete-at-point-functions): corfu

  (use-package corfu
    :ensure t
    :demand t                      ; need this when using :bind or :hook
    :config
    (global-corfu-mode 1)
    :custom
    (corfu-auto t)
    (corfu-auto-delay 0.2)         ;; Pop up sooner (was 0.75 -- a noticeable pause).
    (corfu-auto-prefix 2)          ;; Trigger after 2 characters instead of the default 3.
    (corfu-quit-no-match 'separator) ;; Keep the popup open while typing; only quit on a real dead-end.
    (corfu-min-width 40)           ;; Minimum popup width in columns (default 15) -- wider box.
    (corfu-max-width 140)          ;; Allow a wider box so right-hand annotations aren't clipped.
    ;; Might want to customize corfu-sort-function
    :bind
    (("M-RET" . completion-at-point)
     )
    )

  ;; corfu extension (in corfu/extensions/corfu-history.el); load after corfu
  (use-package corfu-history
    :ensure nil
    :after corfu
    :config
    (corfu-history-mode)
    (savehist-mode 1)
    (add-to-list 'savehist-additional-variables 'corfu-history)
    )

  ;; corfu extension: show full documentation / signature for the highlighted
  ;; candidate in a side popup, so long annotations don't need to fit on the
  ;; candidate line. Toggle the popup manually with M-h while completing.
  (use-package corfu-popupinfo
    :ensure nil
    :after corfu
    :hook (corfu-mode . corfu-popupinfo-mode)
    :custom
    (corfu-popupinfo-delay '(0.5 . 0.2))   ;; (initial . subsequent) seconds before the popup shows.
    (corfu-popupinfo-max-width 100)
    (corfu-popupinfo-max-height 20))

  (use-package corfu-terminal
    :after corfu
    :init
    (defvar corfu-terminal-mode)
    ;; TODO set this up, for use in non-GUI emacs
    )

  ;; Additional capf completion sources
  (use-package cape
    :config
    ;; Note: order matters here. First one returning a result wins. Use
    ;; ~add-hook~ to add these, since it sets the global (default) value
    ;; of capf, instead of ~setq~ which would make it buffer-local
    ;; (which would be bad): capf is automatically buffer-local when
    ;; set.
    ;; The buffer-local value, which takes precedence over these, calls these as long
    ;; as it ends with ~t~.
    (add-hook 'completion-at-point-functions #'cape-history)
    (add-hook 'completion-at-point-functions #'cape-file)
    (add-hook 'completion-at-point-functions #'cape-keyword)
    (add-hook 'completion-at-point-functions #'cape-dabbrev)
    (message (format "Loading my capf extensions: %s" completion-at-point-functions))
    )

  ;; Nice icons for corfu popups
  (use-package kind-icon
    :after corfu
    :custom
    (kind-icon-default-face 'corfu-default)   ;; Derive the icon background from the popup face.
    :config

** Vertico
Vertico enhances the completion experience in Emacs by providing a vertical selection interface for both buffer and minibuffer completions. Unlike traditional minibuffer completion, which displays candidates in a horizontal format, Vertico presents candidates in a vertical list, making it easier to browse and select from multiple options.

In buffer completion, `switch-to-buffer' allows you to select from open buffers. Vertico streamlines this process by displaying the buffer list in a way that improves visibility and accessibility. This is particularly useful when you have many buffers open, allowing you to quickly find the one you need.

In minibuffer completion, such as when entering commands or file paths, Vertico helps by showing a dynamic list of potential completions, making it easier to choose the correct one without typing out the entire string.

#+begin_src emacs-lisp

  (use-package vertico
    :demand t
    :init (vertico-mode)           ;; Enable vertico after Emacs has initialized.
    :custom
    (vertico-count 10)                    ;; Number of candidates to display in the completion list.
    (vertico-resize nil)                  ;; Disable resizing of the vertico minibuffer.
    (vertico-cycle nil)                   ;; Do not cycle through candidates when reaching the end of the list.
    :config
    ;; Customize the display of the current candidate in the completion list.
    ;; This will prefix the current candidate with “» ” to make it stand out.
    ;; Reference: https://github.com/minad/vertico/wiki#prefix-current-candidate-with-arrow
    (advice-add #'vertico--format-candidate :around
                (lambda (orig cand prefix suffix index _start)
                  (setq cand (funcall orig cand prefix suffix index _start))
                  (concat
                   (if (= vertico--index index)
                       (propertize "» " 'face '(:foreground "#80adf0" :weight bold))
                     "  ")
                   cand))))

#+end_src

*** Denote
[[https://protesilaos.com/emacs/denote][Denote]] is a simple, file-based note-taking system. Every note is a plain Org

** LSP (Eglot)
Emacs ships with a built-in LSP client, =eglot=, which is fully integrated with
the native Emacs facilities: completion through =completion-at-point= (so it
works directly with Corfu, with no extra completion backend), diagnostics
through =flymake=, and documentation through =eldoc=. Because it is built in, it
needs no installation (=:ensure nil=) and stays compatible with whatever Emacs
version you run.

NOTE: Eglot does not bundle the language servers themselves — install those with
your system package manager or =npm=:
- Bash:       =bash-language-server=
- Python:     =python-lsp-server= (pylsp) or =pyright=
- JS/TS/TSX:  =typescript-language-server=
  Eglot's docs: https://www.gnu.org/software/emacs/manual/html_mono/eglot.html

  #+begin_src emacs-lisp

    (use-package eglot
      :ensure nil          ;; Built-in since Emacs 29.
      :defer t
      :hook (;; Start eglot automatically in these major modes.
    		 (bash-ts-mode      . eglot-ensure)   ;; Bash
    		 (python-mode       . eglot-ensure)   ;; Python
    		 (python-ts-mode    . eglot-ensure)   ;; Python (tree-sitter)
    		 (typescript-ts-mode . eglot-ensure)  ;; TypeScript
    		 (tsx-ts-mode       . eglot-ensure)   ;; TSX
    		 (js-mode           . eglot-ensure)   ;; JavaScript
    		 (js-ts-mode        . eglot-ensure))  ;; JavaScript (tree-sitter)
      :custom
      (eglot-autoshutdown t)            ;; Shut a server down when its last buffer is killed.
      (eglot-events-buffer-size 0)      ;; Don't log LSP traffic (faster, less memory).
      (eglot-extend-to-xref t)          ;; Allow xref to jump into out-of-project files.
      :config
      ;; Don't let eglot take over the echo area with multi-line docs; keep eldoc
      ;; concise and let `mb/eglot-describe-and-jump' pop the full buffer on demand.
      (setq eldoc-echo-area-use-multiline-p 1))

    (defun mb/eglot-describe-and-jump ()
      "Show hover documentation in the *eldoc* buffer and switch to it."
      (interactive)
      (eldoc-doc-buffer t))

  #+end_src
